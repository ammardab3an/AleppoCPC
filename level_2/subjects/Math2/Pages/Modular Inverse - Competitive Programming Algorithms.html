    Definition

    A modular multiplicative inverse of an integer aa is an integer xx such that a⋅xa⋅x is congruent to 11 modular some modulus mm. To write it in a formal way: we want to find an integer xx so that

    a⋅x≡1modm.a⋅x≡1modm.
    We will also denote xx simply with a−1a−1.

    We should note that the modular inverse does not always exist. For example, let m=4m=4, a=2a=2. By checking all possible values modulo mm is should become clear that we cannot find a−1a−1 satisfying the above equation. It can be proven that the modular inverse exists if and only if aa and mm are relatively prime (i.e. gcd(a,m)=1gcd(a,m)=1).

    In this article, we present two methods for finding the modular inverse in case it exists, and one method for finding the modular inverse for all numbers in linear time.

    Finding the Modular Inverse using Extended Euclidean algorithm

    Consider the following equation (with unknown xx and yy):

    a⋅x+m⋅y=1a⋅x+m⋅y=1

    This is a Linear Diophantine equation in two variables. As shown in the linked article, when gcd(a,m)=1gcd(a,m)=1, the equation has a solution which can be found using the extended Euclidean algorithm. Note that gcd(a,m)=1gcd(a,m)=1 is also the condition for the modular inverse to exist.

    Now, if we take modulo mm of both sides, we can get rid of m⋅ym⋅y, and the equation becomes:

    a⋅x≡1modma⋅x≡1modm

    Thus, the modular inverse of aa is xx.

    The implementation is as follows:

    int x, y;
int g = extended_euclidean(a, m, x, y);
if (g != 1) {
    cout << "No solution!";
}
else {
    x = (x % m + m) % m;
    cout << x << endl;
}

    Notice that we way we modify x. The resulting x from the extended Euclidean algorithm may be negative, so x % m might also be negative, and we first have to add m to make it positive.

    Finding the Modular Inverse using Binary Exponentiation

    Another method for finding modular inverse is to use Euler's theorem, which states that the following congruence is true if aa and mm are relatively prime:

    aϕ(m)≡1modmaϕ(m)≡1modm

    ϕϕ is Euler's Totient function. Again, note that aa and mm being relative prime was also the condition for the modular inverse to exist.

    If mm is a prime number, this simplifies to Fermat's little theorem:

    am−1≡1modmam−1≡1modm

    Multiply both sides of the above equations by a−1a−1, and we get:

    • For an arbitrary (but coprime) modulus mm: aϕ(m)−1≡a−1modmaϕ(m)−1≡a−1modm
    • For a prime modulus mm: am−2≡a−1modmam−2≡a−1modm

    From these results, we can easily find the modular inverse using the binary exponentiation algorithm, which works in O(logm)O(log⁡m) time.

    Even though this method is easier to understand than the method described in previous paragraph, in the case when mm is not a prime number, we need to calculate Euler phi function, which involves factorization of mm, which might be very hard. If the prime factorization of mm is known, then the complexity of this method is O(logm)O(log⁡m).

    Finding the modular inverse for every number modulo mm

    The problem is the following: we want to compute the modular inverse for every number in the range [1,m−1][1,m−1].

    Applying the algorithms described in the previous sections, we can obtain a solution with complexity O(mlogm)O(mlog⁡m).

    Here we present a better algorithm with complexity O(m)O(m). However for this specific algorithm we require that the modulus mm is prime.

    We denote by inv[i]inv[i] the modular inverse of ii. Then for i>1i>1 the following equation is valid:

    inv[i]=−⌊mi⌋⋅inv[mmodi]modminv[i]=−⌊mi⌋⋅inv[mmodi]modm

    Thus the implementation is very simple:

    inv[1] = 1;
for(int i = 2; i < m; ++i)
    inv[i] = m - (m/i) * inv[m%i] % m;
